# AI Resume Platform

一个基于大模型的智能简历解析与岗位匹配平台，帮助企业高效筛选人才。

## 📋 项目简介

本平台集成了豆包大模型（火山引擎），实现简历的智能解析、候选人管理和岗位匹配功能，提供完整的人才招聘流程解决方案。

## 🚀 本地开发环境搭建

### 环境要求

- Node.js >= 18.0.0
- PostgreSQL >= 16.0
- npm >= 9.0.0

### 后端搭建

```bash
# 进入后端目录
cd ai-resume-backend

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入数据库连接和 API Key

# 数据库迁移
npx prisma migrate dev

# 启动服务（开发模式）
npm start
```

后端服务运行在 `http://localhost:8000`

### 前端搭建

```bash
# 进入前端目录
cd ai-resume-platform

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env.local
# 编辑 .env.local 文件，配置后端 API 地址

# 启动开发服务器
npm run dev
```

前端服务运行在 `http://localhost:3000`

## 📦 部署方式

### 前端部署（Vercel）

```bash

# 登录并部署
vercel login

```

### 后端部署（Render）

1. 登录 [Render.com](https://render.com)
2. 创建新的 Web Service
3. 连接 GitHub 仓库
4. 配置：
   - Build Command: `npm install && npx prisma migrate deploy`
   - Start Command: `node server.js`
   - 环境变量配置

### 数据库部署（Supabase）

1. 注册 [Supabase](https://supabase.com)
2. 创建新项目
3. 获取数据库连接字符串
4. 更新后端环境变量：
   ```
   DATABASE_URL="postgresql://postgres:密码@地址:5432/postgres"
   ```

```bash

docker-compose up -d
```

## 🎯 关键技术决策

### 1. SSE 流式输出
- **决策**：简历解析和岗位匹配使用 SSE（Server-Sent Events）
- **理由**：大模型调用耗时较长，流式输出可实时反馈进度，提升用户体验

### 2. 模拟模式降级
- **决策**：当 API Key 无效时自动切换到模拟数据模式
- **理由**：确保开发和测试环境稳定运行，不受大模型服务限制

### 3. 统一响应格式
- **决策**：所有接口返回 `{code, data, msg}` 格式
- **理由**：前端统一处理，简化错误判断逻辑

### 4. 服务端组件分离
- **决策**：Next.js 布局组件分离为纯服务端和客户端组件
- **理由**：避免客户端 Hooks（如 usePathname）与服务端特性（如 metadata）冲突

### 5. 数据库数组类型
- **决策**：技能字段使用 PostgreSQL 原生数组类型
- **理由**：支持高效的数组操作和查询，避免创建中间表

## 📝 开发注意事项

1. **大模型配置**：需在火山引擎控制台获取有效的 API Key
2. **文件上传**：上传的 PDF 文件存储在 `uploads` 目录
3. **环境变量**：敏感信息（API Key、数据库密码）必须通过环境变量配置
4. **CORS**：开发环境已配置允许本地访问，生产环境需配置具体域名
5. **HTTPS**：生产环境必须启用 HTTPS，否则大模型 API 可能无法调用
